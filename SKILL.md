# 课程研发设计 Skill（ADDIE × 大片课融合版）

## 功能说明

本技能实现一套完整的课程研发设计流程，融合两大方法论：

1. **ADDIE体系化框架** — 管流程骨架（分析→设计→开发→实施→评估）
2. **大片课实战方法论** — 管内容质量（天龙八部：定位→框架→开场→中场→收场→共情→悬疑→销售）

核心理念：**课程的本质是「换脑」，唯有大片式课程才能实现深度认知改变**

---

## 全景流程

```
阶段一：分析 × 定位 ──→ 阶段二：设计 × 框架 ──→ 阶段三：开发 × 内容
  (找钉子)                (传信任)               (写剧本)
                                                      │
                                                      ▼
阶段五：评估 × 销售 ←── 阶段四：实施 × 演绎 ←────────┘
 (闭环迭代)              (说动学员)
```

## 阶段一：分析 × 大片定位

### 使命：找到课程的钉子

**钉子公式 = "你有病，我有药，就我有"**

### 执行步骤

#### 1.1 学员调研
- 调研对象：至少10位目标学员的深度访谈或问卷
- 核心问题：
  - 你现在面临的最大挑战是什么？（找痛点）
  - 你期望达到什么水平？（找差距）
  - 你之前尝试过什么方法？为什么没用？（找现状）
  - 你希望学完后能做什么？（找目标）

#### 1.2 找钉子（三条路径选一条）

**路径A：从大痛点出发**
```
定领域 → 定画像 → 定挑战（追问5层为什么） → 定观点
```

**路径B：从大师论出发**
```
定领域 → 定画像 → 定观点（核心方法论） → 定挑战
```

**路径C：从大竞争出发**
```
找最佳实践的独特感悟 → 与主流观点的不同 → 定位
```

#### 1.3 钉子三要素输出

| 要素 | 问题 | 输出格式 |
|------|------|---------|
| 一痛 | 学员最大的痛点？ | 一句话 |
| 一招 | 你的核心解决方案？ | 一句话 |
| 一不 | 与竞争对手的最大不同？ | 一句话 |

#### 1.4 质量检查
- [ ] 钉子能一句话说清
- [ ] 任何人3秒内能理解
- [ ] 同时满足：大师论 + 大挑战 + 大竞争
- [ ] 讲师自己有底气反复强调

> ⚠️ **没有钉子绝不进入下一阶段！**

## 阶段二：设计 × 大片框架

### 使命：设计课程骨架，让框架传递信任

### 执行步骤

#### 2.1 课程架构设计

**拒绝纪录片式，采用大片式：**

| 维度 | 纪录片式 ❌ | 大片式 ✅ |
|------|-----------|----------|
| 结构 | 主题A→讲完→主题B | 故事主线串联所有模块 |
| 组织 | 知识点罗列 | 挑战→解决方案→新挑战→新方案 |
| 感受 | "又来上课了" | "这是大片，我想看下去" |

**大片框架模板：**
```
主线：[一句话钉子]
模块1：[决心改命] → 模块2：[挑战现状] → 模块3：[促使行动]
  开场定调          中场深挖          收场落地
```

#### 2.2 三把利剑检验

| 检验 | 问题 | 通过标准 |
|------|------|---------|
| 一脉相承 | 每个模块与钉子的关系清楚吗？ | 随机能说清 |
| 一剑封喉 | 去掉任一模块课程是否不完整？ | 每个都不可少 |
| 一首诗词 | 框架朗朗上口吗？ | 押韵/对仗/节奏感 |

#### 2.3 1/3/5分钟检验

| 时间 | 标准 | 检验方法 |
|------|------|---------|
| 1分钟 | 入眼 | 给人看框架图，一看就被吸引 |
| 3分钟 | 入耳 | 口头讲解框架，一听就被打动 |
| 5分钟 | 入心 | 对方能自己复述并转介绍 |

#### 2.4 评估方案设计
- 前测：课前认知水平测验
- 形成性评估：每个模块结束的小测验/实操
- 总结性评估：课程结束的综合考核/项目
- 跟踪评估：课后1-3个月行为改变追踪

## 阶段三：开发 × 大片内容

### 使命：把设计稿变成有血有肉的课程内容

### 执行步骤

#### 3.1 大片开场开发（决心改命）

每个教学单元的开场三步曲：

| 步骤 | 内容 | 动作 |
|------|------|------|
| 定义使命 | 学员要实现的最高使命 | 抛出核心问题，解释为什么重要 |
| 定义愿景 | 实现后的美好画面 | 故事/案例/数据展现"做成了会怎样" |
| 定义标准 | 重新定义行业高标准 | "你要做到这个水平才叫达标" |

**开场模板：**
```
"你面对的最大使命是 [使命]。
 如果做到了，你将 [愿景]。
 但现实的标准是 [高标准]，你现在离这个标准还有 [差距]。
 今天这堂课，我们一起解决这个问题。"
```

#### 3.2 大片中场开发（挑战现状）

| 步骤 | 内容 | 动作 |
|------|------|------|
| 揭示现状 | 具体描述旧认知/旧做法 | 案例/数据/对比 |
| 挑战现状 | 分析旧做法为何失败 | 追问到根本原因 |
| 标杆对比 | 最佳实践与现状的差距 | 用自身最佳实践案例 |

**中场模板：**
```
"大多数人的做法是 [现状]。
 但这带来了一个问题： [具体问题]。
 为什么会这样？因为 [根本原因]。
 而最佳实践告诉我们 [标杆对比]。"
```

#### 3.3 大片收场开发（促使行动）

| 步骤 | 内容 | 动作 |
|------|------|------|
| 抓牛鼻子 | 核心一招中最关键的执行点 | 聚焦1个绝招 |
| 做标准 | 大概念→可操作动作 | 落实到责任人、时间、可检查 |
| 给信心 | 预期+希望+决心+勇气 | 亲身经历/学员成功案例 |

**收场模板：**
```
"解决方案就是 [绝招]。
 具体怎么做？第一步 [动作1]，第二步 [动作2]...
 我自己 [亲身经历/案例]，你们一定也可以。
 现在就去做。"
```

#### 3.4 全场共情设计

| 类型 | 方法 | 应用 |
|------|------|------|
| 身份共情 | "我曾经和你一样…" | 开场建立信任 |
| 情感共情 | 用学员的语言讲学员的故事 | 中场引发共鸣 |
| 语言共情 | 大白话，避免术语 | 全程贯穿 |
| 状态共情 | 巅峰状态、充满激情 | 全场演绎 |

#### 3.5 全场悬疑设计

| 手法 | 方法 | 应用 |
|------|------|------|
| 抛问题 | 提出让学员好奇的问题 | 每个模块开场 |
| 给挑战 | 设置需要思考的难题 | 中场深入 |
| 揭答案 | 最佳时机揭晓方案 | 收场关键 |

#### 3.6 90分钟课堂节奏模板

```
┌─────────────┬───────────┬───────────────┐
│  开场 15min  │ 中场 50min │  收场 25min   │
├─────────────┼───────────┼───────────────┤
│ ·定义使命   │ ·揭示现状  │ ·抓牛鼻子     │
│ ·定义愿景   │ ·挑战现状  │ ·做标准       │
│ ·定义标准   │ ·标杆对比  │ ·给信心       │
├─────────────┴───────────┴───────────────┤
│ 全程贯穿：共情 + 悬疑                      │
└─────────────────────────────────────────┘
```

## 阶段四：实施 × 课程演绎

### 使命：把课程交付给学员，说动他们去行动

### 执行步骤

#### 4.1 实施准备清单
- [ ] 讲师磨课完成（重点训练共情和悬疑）
- [ ] 技术环境/场地/设备就绪
- [ ] 学员通知和学习指南发出
- [ ] 试点运行完成并调整

#### 4.2 每节课演绎自检
- [ ] 开场让学员下定决心了吗？
- [ ] 中场成功挑战学员现状了吗？
- [ ] 收场学员有可执行的行动方案吗？
- [ ] 学员感受到"你懂我"了吗？
- [ ] 学员全程保持投入和期待吗？

## 阶段五：评估 × 大片销售

### 使命：验证效果、驱动迭代、实现口碑传播

### 执行步骤

#### 5.1 双重评估体系

**ADDIE柯氏四级评估：**
| 层级 | 内容 | 时机 |
|------|------|------|
| L1 反应 | 满意度、参与度 | 课程结束 |
| L2 学习 | 知识/技能掌握 | 课程中/结束 |
| L3 行为 | 行为改变应用 | 课后1-3月 |
| L4 结果 | 业务绩效影响 | 课后3-6月 |

**大片课换脑八问评估：**
1. 学员能一句话说清课程内容吗？（定位）
2. 学员能复述课程框架吗？（框架）
3. 学员听完开场有改变决心吗？（开场）
4. 学员承认旧认知被挑战了吗？（中场）
5. 学员有具体的行动方案吗？（收场）
6. 学员觉得"讲师懂我"吗？（共情）
7. 学员全程投入吗？（悬疑）
8. 学员愿意推荐/复购吗？（销售）

#### 5.2 钉子营销（不销而销）

| 步骤 | 方法 |
|------|------|
| 定义价值 | 课程中不断强化钉子的价值 |
| 承诺价值 | 用成果和案例兑现承诺 |
| 传播价值 | 让学员自发传播、转介绍 |

#### 5.3 数据驱动迭代

```
评估数据 → 问题诊断 → 回溯对应阶段 → 修改 → 再评估

L1低 → 回到Design（框架、策略、媒体）
L2低 → 回到Development（内容质量、练习）
L3低 → 回到Implementation（支持体系、实践）
L4低 → 回到Analysis（需求是否准确）
换脑八问不达标 → 回到大片课对应工序
```

## TTT（Training the Trainer）核心要点

### 讲师能力三层模型

```
第一层：专业功底（你讲什么）
  → 你在这个领域的深度洞察和实践经验
  → 没有专业功底，技术再好也是空壳

第二层：授课技术（你怎么讲）
  → 大片天龙八部 = 八道工序
  → 共情 + 悬疑的演绎能力

第三层：个人魅力（你是谁）
  → 巅峰状态、信念感、感染力
  → 学员不仅学你的知识，更要成为你
```

### 讲师常见问题与对策

| 问题 | 根因 | 对策 |
|------|------|------|
| 说不清定位 | 没有钉子/钉子太多 | 找钉子，取一舍九 |
| 讲不深刻 | 没有追问到根本原因 | 追问5层为什么 |
| 讲太多招 | 不敢做减法 | 抓主要矛盾，只讲一招 |
| 没有共情 | 只关注内容不关注人 | 用学员的语言讲学员的故事 |
| 没有悬疑 | 平铺直叙 | 每个模块抛问题→给挑战→揭答案 |
| 不敢卖课 | 怕损害专家形象 | 钉子营销，不销而销 |

### 磨课（课程打磨）SOP

```
Step 1: 听完整课程（全程录音/录像）
Step 2: 用换脑八问逐项评估
Step 3: 定位诊断 — 钉子清楚吗？
Step 4: 框架诊断 — 三把利剑通过吗？
Step 5: 开场诊断 — 能让人决心改命吗？
Step 6: 中场诊断 — 敢挑战学员现状吗？
Step 7: 收场诊断 — 有行动方案和信心吗？
Step 8: 共情诊断 — 学员觉得懂他吗？
Step 9: 悬疑诊断 — 学员全程投入吗？
Step 10: 输出修改建议 → 讲师修改 → 再磨
```

## AI辅助课程开发

### 各阶段AI赋能点

| 阶段 | AI辅助方式 | 推荐工具 |
|------|-----------|---------|
| 分析 | AI辅助学员调研数据分析、画像生成 | ChatGPT/Claude |
| 设计 | AI生成课程框架初稿、大纲建议 | ChatGPT/Claude |
| 开发 | AI写脚本、生成案例、制作课件图 | ChatGPT+Midjourney+Canva |
| 实施 | AI数字人授课、智能答疑 | HeyGen/D-ID |
| 评估 | AI自动收集和分析学习数据 | LMS平台+AI分析 |

### AI漫剧培训特殊考量

| 维度 | 建议 |
|------|------|
| 钉子定位 | "用AI从0到1出一部漫剧" — 痛点明确 |
| 课程框架 | 以真实项目为主线，非工具操作罗列 |
| 内容开发 | 用AI工具本身制作课程素材（用AI教AI） |
| 评估方式 | 以学员产出的漫剧作品为核心评估 |
| 共情设计 | "我也曾经踩过这些坑" — 讲师的AI创作血泪史 |

## 使用方法

### 启动课程研发

```
用户：我要开发一门 [主题] 课程，目标学员是 [画像]，想解决 [问题]
```

Agent应：
1. 复述理解用户需求
2. 进入阶段一：协助找钉子
3. 引导用户完成钉子三要素（一痛/一招/一不）
4. 钉子确定后进入阶段二

### 现有课程打磨

```
用户：我有一门 [课程名]，帮我打磨升级
```

Agent应：
1. 要求用户提供现有课程大纲/PPT/录像
2. 用换脑八问逐项诊断
3. 输出问题清单和修改建议
4. 协助修改对应阶段

### 课程评审/诊断

```
用户：帮我评审一下这门课的质量
```

Agent应：
1. 收集课程材料（大纲/PPT/脚本/录像文字稿）
2. 按换脑八问逐项打分
3. 按柯氏四级预期输出评估维度
4. 输出诊断报告和改进建议
