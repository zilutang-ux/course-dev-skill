# ADDIE五阶段详细指南

> ADDIE是全球最通用的教学设计模型（ISD），源于1975年美国军方。
> 五个阶段：Analysis（分析）→ Design（设计）→ Development（开发）→ Implementation（实施）→ Evaluation（评估）

## Phase 1: Analysis（分析）

**核心问题：** 解决什么问题？给谁学？学什么？

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| 需求分析 | 识别绩效差距，确定培训是否是最佳解决方案 | 需求分析报告 |
| 学习者分析 | 受众画像：知识基础、学习动机、技术能力、学习环境 | 学习者画像文档 |
| 目标定义 | 明确学习目标（知识/技能/态度三个维度） | 学习目标清单 |
| 约束识别 | 时间、预算、技术限制、组织因素 | 约束条件清单 |
| 内容范围 | 确定知识点边界和优先级 | 内容大纲/范围文档 |
| 交付方式 | 线上/线下/混合/移动端等 | 交付策略建议 |

### 检查清单
- [ ] 完成学习需求调研（访谈/问卷/数据分析）
- [ ] 明确绩效差距和培训目标
- [ ] 完成学习者画像（背景、动机、约束）
- [ ] 确定学习目标（知识/技能/态度）
- [ ] 评估技术与资源约束
- [ ] 确定交付方式和平台

---

## Phase 2: Design（设计）

**核心问题：** 怎么教？怎么学？怎么测？

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| 学习目标细化 | 按布鲁姆分类法制定可测量目标 | 细化目标矩阵 |
| 教学策略选择 | 根据内容类型选择策略 | 教学策略文档 |
| 课程架构设计 | 模块→单元→课时的层级结构 | 课程结构图/大纲 |
| 评估方案设计 | 前测、形成性评估、总结性评估 | 评估方案 |
| 媒体与资源规划 | 视频/图文/互动/VR等媒体选择 | 媒体矩阵 |
| 故事板/脚本框架 | 每个学习单元的内容流和交互逻辑 | 故事板模板 |
| 时间线规划 | 各模块学习时长分配 | 项目时间表 |

### 布鲁姆认知目标分类（修订版）
```
创造 (Create)     ← 最高层
评价 (Evaluate)
分析 (Analyze)
应用 (Apply)
理解 (Understand)
记忆 (Remember)    ← 最低层
```

### 教学策略匹配参考
| 内容类型 | 推荐策略 |
|----------|----------|
| 事实性知识 | 闪卡、测验、信息图 |
| 概念性知识 | 类比、对比、案例分析 |
| 流程性知识 | 演示、模拟、步骤清单 |
| 原理性知识 | 情境案例、问题解决、讨论 |
| 社会性技能 | 角色扮演、协作项目、反馈 |

### 检查清单
- [ ] 学习目标按布鲁姆分类细化
- [ ] 课程结构/大纲确定
- [ ] 教学策略选定并匹配内容类型
- [ ] 评估方案设计完成
- [ ] 媒体和资源类型确定
- [ ] 故事板或脚本框架完成
- [ ] 项目时间表和里程碑确定

## Phase 3: Development（开发）

**核心问题：** 把设计稿变成可交付的学习产品

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| 内容创作 | 撰写脚本、制作课件、录制视频/音频 | 课程内容素材 |
| 媒体制作 | 图形设计、动画制作、交互开发 | 多媒体素材 |
| 平台搭建 | LMS部署、课程上传、功能配置 | 可运行的课程 |
| 评估工具开发 | 试题编写、量规设计、作业模板 | 评估工具包 |
| 内部评审 | SME审核内容准确性 | 评审记录 |
| 测试调试 | 功能测试、兼容性测试、用户体验测试 | 测试报告 |

### 检查清单
- [ ] 所有课程内容创作完成
- [ ] 多媒体素材制作完成
- [ ] 评估工具开发完成
- [ ] SME审核通过
- [ ] 功能和兼容性测试通过
- [ ] 用户体验测试通过

---

## Phase 4: Implementation（实施）

**核心问题：** 让学习者开始学起来

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| 培训师准备 | 培训师培训（TTT） | 培训师手册 |
| 学习者准备 | 技术准备、学习指南、期望管理 | 学习者指南 |
| 环境部署 | 技术环境配置、账号开通、权限设置 | 部署确认清单 |
| 试点运行 | 小规模试运行，收集反馈 | 试点报告 |
| 正式发布 | 全面铺开课程 | 上线公告 |
| 学习支持 | 讨论区、答疑机制、技术支持 | 支持系统 |

### 实施模式对比
| 模式 | 适用场景 | 注意事项 |
|------|----------|----------|
| 线下集中培训 | 技能类、团队建设 | 场地、设备、人员调度 |
| 纯在线异步 | 知识类、大规模覆盖 | 自律性要求高 |
| 直播教学 | 需要实时互动 | 时间协调、技术稳定性 |
| 混合式学习 | 复杂技能、长周期 | 线上线下衔接设计 |
| 翻转课堂 | 深度讨论、应用导向 | 预习完成率是关键 |

### 检查清单
- [ ] 培训师培训完成
- [ ] 学习者指南和准备材料就绪
- [ ] 技术环境部署完成
- [ ] 试点运行完成并收集反馈
- [ ] 正式发布
- [ ] 学习支持机制建立

## Phase 5: Evaluation（评估）

**核心问题：** 学到了吗？有效果吗？怎么改进？

### 柯氏四级评估模型

| 层级 | 评估内容 | 评估方法 | 时机 |
|------|----------|----------|------|
| **L1 反应** | 学习者满意度、参与度 | 问卷、即时反馈 | 课程结束时 |
| **L2 学习** | 知识/技能掌握程度 | 前后测对比、实操考核 | 课程中/结束时 |
| **L3 行为** | 所学是否应用于工作 | 跟踪观察、360反馈、绩效数据 | 课程后1-3个月 |
| **L4 结果** | 对组织绩效的影响 | 业务KPI对比、ROI分析 | 课程后3-6月 |

### 评估数据驱动改进
```
评估数据 → 问题诊断 → 回溯到对应阶段 → 修改 → 再评估

L1低 → 检查Design（教学策略、媒体选择）
L2低 → 检查Development（内容质量、练习设计）
L3低 → 检查Implementation（支持体系、实践机会）
L4低 → 检查Analysis（需求是否准确、目标是否对齐）
```

### 检查清单
- [ ] L1满意度评估完成
- [ ] L2学习效果评估完成（前后测对比）
- [ ] L3行为改变追踪计划制定
- [ ] L4业务结果评估数据收集
- [ ] 综合评估报告撰写
- [ ] 改进建议和迭代计划制定

## ADDIE vs 进化模型

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| **SAM** | 快速原型+迭代，敏捷化ADDIE | 快速开发、互联网产品 |
| **Dick & Carey** | 更细化的系统化步骤 | 学术课程、标准化培训 |
| **Kemp** | 以学习者为中心的环形设计 | 定制化培训 |
| **Agile + ADDIE** | 敏捷开发与ADDIE结合 | AI时代快节奏课程开发 |
| **Rapid Prototyping** | 快速原型→测试→迭代 | 紧急项目、MVP |
